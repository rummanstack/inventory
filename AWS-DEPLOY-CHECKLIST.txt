BEFORE YOU START
----------------
Your database: Supabase (already works, no changes needed)
Your code:     Ready (lambda.js, template.yaml, samconfig.toml all done)


STEP 1 — Create AWS Account
----------------------------
1. Go to https://aws.amazon.com
2. Sign up (needs a credit card, but free tier covers you)
3. Log in to AWS Console


STEP 2 — Create IAM User (for CLI access)
------------------------------------------
1. In AWS Console, search "IAM" and open it
2. Click Users → Create User
3. Username: stockledger-deploy
4. Click Next → Attach policies directly
5. Search and select: AdministratorAccess
6. Click Next → Create User
7. Open the user → Security credentials tab
8. Click "Create access key"
9. Choose: Command Line Interface (CLI)
10. Download the CSV file — save it safely


STEP 3 — Install SAM CLI
--------------------------
Windows download:
https://docs.aws.amazon.com/serverless-application-model/latest/developerguide/install-sam-cli.html

After install, verify:
  sam --version


STEP 4 — Install AWS CLI
--------------------------
Windows download:
https://awscli.amazonaws.com/AWSCLIV2.msi

After install, run:
  aws configure

Enter:
  AWS Access Key ID:     (from the CSV in Step 2)
  AWS Secret Access Key: (from the CSV in Step 2)
  Default region:        ap-south-1
  Default output format: json


STEP 5 — Update samconfig.toml
--------------------------------
Open: samconfig.toml (in your project root)

Replace the DatabaseUrl line with your real Supabase URL:

  "DatabaseUrl=postgresql://postgres.lwerihbldpyuzciyqvmh:s1PcUrahMBgQ72TE@aws-1-ap-northeast-1.pooler.supabase.com:6543/postgres?sslmode=no-verify&pgbouncer=true&connection_limit=1&schema=public",

Save the file.


STEP 6 — Deploy Backend to Lambda
-----------------------------------
Open terminal in your project root folder, then run:

  sam build

Wait for it to finish, then run:

  sam deploy

It will ask a few questions — just press Enter for each (defaults are fine).
At the end it prints your API URL. Copy and save it.
Looks like: https://xxxxxxxxxx.execute-api.ap-south-1.amazonaws.com


STEP 7 — Build Frontend
------------------------
Run:
  npm --prefix frontend run build

This creates the frontend/dist folder.


STEP 8 — Create S3 Bucket
---------------------------
Run:
  aws s3 mb s3://stockledger-frontend --region ap-south-1


STEP 9 — Upload Frontend to S3
--------------------------------
Run:
  aws s3 sync frontend/dist s3://stockledger-frontend --delete


STEP 10 — Set Up CloudFront
-----------------------------
1. In AWS Console, search "CloudFront" and open it
2. Click "Create Distribution"
3. Origin domain: select your S3 bucket
4. Default root object: index.html
5. Click "Create Distribution"
6. Wait for status to become "Deployed" (takes ~5 minutes)
7. Copy your CloudFront domain (like d1abc123.cloudfront.net)

Then add a second origin for the API:
1. Open your distribution → Origins tab → Create origin
2. Origin domain: paste your API Gateway URL from Step 6
3. Save

Then add a behavior for API routes:
1. Behaviors tab → Create behavior
2. Path pattern: /api/*
3. Origin: select your API Gateway origin
4. Allowed HTTP methods: GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE
5. Save


STEP 11 — Test Everything
---------------------------
Open your CloudFront URL in a browser.
Try logging in.
If login works, everything is working.


STEP 12 — Update CORS (before sharing with real users)
-------------------------------------------------------
Open template.yaml, find AllowOrigins and change:
  - '*'
to:
  - 'https://your-cloudfront-domain.cloudfront.net'

Then run:
  sam deploy

------------------------

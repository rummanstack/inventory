AWS SES — Forgot Password Email Integration Guide
=================================================
Last updated: 2026-06-28

WHAT ALREADY EXISTS
-------------------
- password_reset_tokens table + repository (create / find / mark-used)
- authService.forgotPassword() creates the token but NEVER emails it — returns {ok:true} silently
- authService.resetPassword() consumes the token
- /api/auth/forgot-password and /api/auth/reset-password routes are live

So we only need to: build an email sender and call it inside forgotPassword().


STEP 1 — VERIFY A SENDER IN AWS SES
-------------------------------------
1. Go to AWS Console → Simple Email Service (SES)
2. In the left panel: "Verified identities" → "Create identity"
3. Choose "Domain" (recommended) or "Email address"
   - Domain: e.g. stockledger.com — AWS gives you DNS records (CNAME/TXT).
     Add those records to your domain's DNS (in Namecheap / Cloudflare etc.).
     Verification takes a few minutes.
   - Email address: simpler — AWS sends a verification link to that address.
     Good enough for testing. Use something like noreply@yourdomain.com.
4. Wait until status shows "Verified".

SANDBOX vs PRODUCTION
  By default SES is in Sandbox mode. In sandbox:
  - You can only send TO verified email addresses (not any user email).
  - To go production: SES → Account Dashboard → "Request production access"
  - Fill the form (use case: transactional, describe reset password emails).
  - AWS approves within 24h usually.
  - During development, just verify a few test email addresses manually.


STEP 2 — CREATE AN IAM USER WITH SES PERMISSION
-------------------------------------------------
1. Go to AWS Console → IAM → Users → "Create user"
2. Name it something like "stockledger-ses"
3. Attach permissions directly → search for "AmazonSESFullAccess"
   (or create a custom policy with only ses:SendEmail if you want minimal scope)
4. After creating the user → "Security credentials" tab → "Create access key"
5. Choose "Application running outside AWS"
6. Download / copy the Access Key ID and Secret Access Key — you won't see the secret again


STEP 3 — ADD ENV VARS TO backend/.env
---------------------------------------
Add these lines to your backend/.env file:

  AWS_ACCESS_KEY_ID=AKIA...your_key...
  AWS_SECRET_ACCESS_KEY=your_secret_key_here
  AWS_SES_REGION=ap-southeast-1
  SES_FROM_EMAIL=noreply@yourdomain.com
  SES_FROM_NAME=StockLedger
  APP_URL=https://yourdomain.com

  AWS_SES_REGION: pick the region where your SES is set up.
    Common choices:
      us-east-1      (N. Virginia — largest quota)
      ap-southeast-1 (Singapore — good for South/Southeast Asia)
      eu-west-1      (Ireland)
    Use the SAME region where you verified your identity in SES.

  APP_URL: the public base URL of your app (no trailing slash).
    Locally: http://localhost:5173
    Production: https://yourdomain.com
    The reset link will be: APP_URL + /reset-password?token=<token>


STEP 4 — INSTALL THE AWS SDK PACKAGE
--------------------------------------
Run this from the backend folder:

  npm install @aws-sdk/client-ses

This is the v3 modular AWS SDK — small bundle, ESM-compatible.

COMMON ERRORS
--------------
MessageRejected: Email address is not verified
  → You're in sandbox. Verify the recipient email in SES console.

InvalidClientTokenId / AuthFailure
  → Wrong AWS_ACCESS_KEY_ID or AWS_SECRET_ACCESS_KEY in .env

SignatureDoesNotMatch
  → AWS_SECRET_ACCESS_KEY has a copy-paste error (watch for trailing spaces)

Sending to unverified addresses in sandbox
  → Request production access or verify the test recipient email

Token not found / expired
  → PASSWORD_RESET_TOKEN_TTL_MINUTES is 60 by default — link expires in 1 hour

Deploy and Submit Indexing Signals

1. Deploy the latest version
- Put the new site live on: https://stockledger.pro
- Make sure the site opens correctly
- Make sure HTTPS is working

2. Check these URLs after deploy
- https://stockledger.pro/robots.txt
- https://stockledger.pro/sitemap.xml
- https://stockledger.pro/landing
- https://stockledger.pro/features
- https://stockledger.pro/solutions
- https://stockledger.pro/software

3. Check page source on a few important pages
Make sure these exist:
- title
- meta description
- canonical URL
- no accidental noindex

Check these pages first:
- /landing
- /features/accounting
- /software/inventory-software-bangladesh
- /pricing
- /contact

4. Submit sitemap in Google Search Console
Submit:
- https://stockledger.pro/sitemap.xml

5. Inspect important URLs in Search Console
Check and request indexing for:
- /landing
- /features
- /solutions
- /software
- /software/accounting-software-bangladesh
- /software/inventory-software-bangladesh
- /pricing
- /contact

6. Make sure private pages are not indexed
These should not be indexed:
- /login
- /register
- dashboard pages
- admin pages

7. Monitor Search Console after deploy
Watch for:
- indexing errors
- blocked pages
- canonical problems
- Core Web Vitals issues

Simple rule
- deploy
- check robots and sitemap
- submit sitemap
- inspect key URLs
- request indexing
- monitor results

# Inventory Product Redesign 7-Day Plan

This plan is for making the app feel premium, modern, and ready for an American market audience.

Recommended implementation direction:
- keep Tailwind as the styling base
- keep custom components for business-specific workflows
- use a lightweight component library only for primitives like dialogs, tabs, dropdowns, tooltips, and date pickers
- prefer `shadcn/ui` style components over a heavy opinionated library

### Day 1: Design Direction and System Setup
Goal:
Define the visual language of the product.

Tasks:
- choose the final color palette
- define typography hierarchy
- set spacing, radius, shadow, and border rules
- define card, table, button, badge, and modal styles
- decide which primitives should come from a component library and which should stay custom
- create a small reusable design system for the app

Deliverable:
- a consistent visual foundation for all screens

### Day 2: Navigation and Shell Redesign
Goal:
Make the app shell feel clean and premium.

Tasks:
- redesign the sidebar
- refine the top bar
- improve user profile and logout area
- simplify active and hover states
- make navigation feel calm, dense, and professional

Deliverable:
- polished app shell with stronger hierarchy

### Day 3: Dashboard and Core Page Layouts
Goal:
Upgrade the most visible pages first.

Tasks:
- redesign dashboard cards and summary areas
- refine table headers and filter bars
- improve page titles, subtitles, and spacing
- make core actions easier to scan
- reduce visual noise across main screens

Deliverable:
- dashboard and master pages feel modern and balanced

### Day 4: Settlement, Finance, and Report Pages
Goal:
Make financial pages feel precise and trustworthy.

Tasks:
- refine settlement page layout
- refine expense, DSR finance, month-end summary, and activity log pages
- improve numeric hierarchy and summary presentation
- make tables easier to read and compare
- keep calculation outputs visually prominent

Deliverable:
- finance-heavy pages look cleaner and more credible

### Day 5: Loading, Empty, Error, and Confirmation States
Goal:
Make the product feel production-grade.

Tasks:
- improve skeleton loaders
- improve empty states
- improve error states
- refine confirmation dialogs
- make all async states feel intentional

Deliverable:
- smoother and more professional interaction feedback

### Day 6: Responsive and Density Refinement
Goal:
Make the app work well on real business screens.

Tasks:
- refine tablet and smaller laptop layouts
- check table overflow behavior
- improve spacing on narrow screens
- preserve usability for operators and managers
- make the app feel stable on different screen sizes

Deliverable:
- responsive behavior is reliable and business-friendly

### Day 7: Final Polish and Market Readiness Pass
Goal:
Remove visual rough edges and prepare for a stronger market presence.

Tasks:
- review every major page for consistency
- fix mismatched spacing, colors, and labels
- check English wording for the American market
- tighten branding details
- review overall product impression as a whole

Deliverable:
- a more premium, cohesive inventory product ready for the next growth step
